# Documentation

This folder contains project documentation.

## Structure
- `overview/` — high-level project information (goals, non-goals, glossary)
- `architecture/` — system design, ADRs, diagrams
- `design/` — (optional) modules, API, data design, UX notes
- `operations/` — (optional) deployment/monitoring/runbooks
- `research/` — (optional) experiments and findings

## Rules
- Keep docs actionable and easy to scan.
- Update docs when behavior or architecture changes.
- Record significant decisions as ADRs under `architecture/adr/`.
