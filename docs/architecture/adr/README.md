# ADRs (Architecture Decision Records)

ADRs capture significant decisions and their trade-offs.

## Rules
- Create ADRs for non-trivial decisions (architecture, security, major dependencies).
- Keep them short and factual.
- Update `index.md` when adding a new ADR.

## Naming
- `ADR-0001-title.md`
- `ADR-0002-title.md`
